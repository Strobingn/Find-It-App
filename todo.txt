IMPORTANT

You need a Firebase project with the Gemini API enabled. Steps:

Go to console.firebase.google.com → Create a project (or use an existing one)
Add an Android app with package com.aistudio.lidardetector.pkrxtz
Enable "AI Logic" (Gemini API) in the Firebase console under Build → AI Logic
Download google-services.json and drop it into app/
The build will compile without it (thanks to passthrough=true), but AI features will show "Firebase not configured" until it's added.

Proposed Features
Feature 1: 🔬 AI Terrain Analyst
What it does: Takes the current hillshade bitmap and sends it to Gemini as a multimodal prompt. Gemini analyzes the terrain image for:

Linear features (foundations, walls, roads, ditches, trenches)
Circular/oval anomalies (wells, pits, burial mounds, gun emplacements)
Geometric patterns suggesting human modification
Natural vs. anthropogenic classification
Confidence level and recommended next steps (where to dig, what to look for)
User flow:

User loads terrain, adjusts visualization (local relief, slope, disturbance mode, etc.)
Taps "🤖 AI Analyze" button on the terrain toolbar
App captures the current hillshadeBitmap + context metadata (grid size, cell size, viz mode, coordinates if available)
Sends to Gemini as content { image(bitmap); text(systemPrompt + context) }
Response streams in real-time into a bottom sheet with markdown-style text
User can tap "📌 Pin finding" to save identified features as TargetSignal entries with source = AI_TERRAIN_SCAN
System prompt: Instructs Gemini to act as an archaeological remote sensing specialist analyzing LiDAR-derived terrain models, using domain-specific vocabulary (local relief model, slope-break, micro-topography, residual anomaly, etc.).

Feature 2: 🎯 Smart Anomaly Scanner
What it does: Combines the existing disturbance-candidate heuristic (visualization mode 5: residual + curvature + roughness scoring) with Gemini classification:

Runs the existing ElevationGrid analysis to compute per-cell disturbance scores
Clusters adjacent high-scoring cells into discrete anomaly regions using connected-component labeling
Extracts a cropped bitmap of each region (with some surrounding context)
Sends the top N regions (≤8) to Gemini in a single batch prompt
Gemini returns structured classifications per region: Foundation, Ditch/Trench, Pit/Well, Mound, Road/Path, Natural erosion, Modern disturbance, Uncertain
User flow:

User is in disturbance-candidate mode (viz mode 5), adjusts sensitivity
Taps "🎯 Classify" button
Loading spinner → results appear as labeled pins on the terrain
Each pin saved as TargetSignal with source = AI_TERRAIN_SCAN and AI notes
Feature 3: 💬 AI Field Assistant Chat
What it does: A contextual chatbot that knows everything about the current session:

Current grid dimensions, cell size, CRS/coordinates
Current visualization mode and all settings (azimuth, altitude, contrast, etc.)
All logged signals and their metadata
Can "see" the hillshade bitmap when asked
Example queries:

"What am I looking at?" → Describes visible terrain features
"Where should I dig test pits?" → Suggests locations based on anomaly patterns
"Is this natural or man-made?" → Analyzes current view
"What historical period could these be from?" → Contextual analysis
"Optimize my lighting for finding walls" → Suggests sun azimuth/altitude
"Generate a summary report" → Text report of all findings
User flow:

FAB button (💬) opens a chat bottom sheet
Quick-action chips at the top: "What's here?", "Find features", "Suggest settings"
User types or taps a chip → response streams in real-time
Multi-turn conversation with full session context
